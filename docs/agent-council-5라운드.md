## 1. CEO (의사결정자): "조건부 GO — 3가지 철칙과 함께"

### 최종 결정: **GO**

#### 4가지 GO 사유

1. **이미 30-35% 자산이 축적됨**: 6개 대학 170개 학과 데이터, Next.js 16 웹앱 동작 중, 비교 기능 구현 완료. 0에서 시작하는 프로젝트가 아니다.
2. **실패 비용이 사실상 0원**: Vercel 무료, 도메인 1-2만원/년, 유일한 비용은 본인 시간
3. **타이밍 윈도우 존재**: 4월 말까지 8주, 5월 과목선택 수요조사 직전
4. **차별화 포인트 명확**: 역방향 검색(내 과목 → 지원 가능 대학)은 현존하는 어떤 서비스에도 없음

### CEO의 3가지 철칙

> **1. 범위는 9개 대학, 역방향 검색 단일 기능으로 고정한다.** "이것도 넣으면 좋겠다"는 유혹이 올 때마다 이 문서를 다시 읽는다.

> **3. 매주 일요일에 KPI 3개를 기록한다.** 감각이 아니라 숫자로 판단한다.

## 3. Mentor (멘토): "완성된 평범함이 미완성된 완벽함을 이긴다"

### 가장 위험한 순간: 런칭 후 2주차

> 런칭 직후 흥분이 사라지고, DAU가 한 자릿수로 떨어질 때 대부분의 사이드 프로젝트가 죽는다. 코드가 아니라 **개발자의 마음**이 먼저 죽는 것이다.

### 실전 처방

| 함정 | 증상 | 처방 |
|------|------|------|
| GA 중독 | 하루 5번+ Analytics 확인 | **주 1회 일요일 저녁**으로 제한 |
| 비교 함정 | "진학사는 벌써 이런 기능도..." | 경쟁사 분석은 **월 1회 30분** 고정 |
| 공허한 빌드 | 아무도 안 쓰는 기능 만드는 느낌 | 2주 1회 **실사용자 1명에게 직접 보여주기** |
| 완벽주의 도피 | "아직 이것도 안 됐는데 홍보하면..." | 부끄럽지 않은 버전은 이미 늦은 버전 |

### 시간 관리: 30분 규칙 + 5분 규칙

- **30분 규칙**: 모든 작업을 30분 단위로 쪼갠다. "대학 데이터 입력"이 아니라 "서울대 자연과학대 3개 학과 입력"
- **5분 규칙**: 기술적 의사결정에 5분 이상 고민하지 않는다. 답이 안 나오면 더 단순한 옵션 선택

### 커뮤니티 신뢰 구축 3단계

| 단계 | 시기 | 행동 | 절대 금지 |
|------|------|------|----------|
| 1단계 | 런칭 전 4주 | 맘카페에서 질문에 성심성의껏 답변. **서비스 언급 금지** | "저희 서비스 써보세요!" |
| 2단계 | 런칭 직후 2주 | "이런 도구를 만들고 있는데 **의견을 구하는** 형태"로 언급 | 댓글마다 링크 넣기 |
| 3단계 | 런칭 후 1개월+ | 답변의 일부로 자연스럽게 링크 포함 | 전문가인 척하기 |

### 실패해도 남는 것

| 얻는 것 | 재사용 가능성 |
|---------|-------------|
| 풀스택 프로젝트 포트폴리오 1개 | 높음 |

## 4. Final Devil's Advocate (최종 악마의 변호인): "자기기만을 멈춰라"

### 도전 1: "실패 비용 0원"은 거짓말이다

| 대안 활용 | 109시간으로 가능한 것 | 기대 가치 |
|-----------|----------------------|----------|
| 프리랜싱 | 시급 3-5만원 x 109시간 | **327-545만원 확정 수입** |
| 기술 학습 | Rust/Go/K8s 중급 과정 | 이직 시 연봉 500-1000만원 상승 |
| 오픈소스 기여 | 유명 프로젝트 PR 10-20개 | 포트폴리오 + 네트워킹 |
| 다른 사이드 프로젝트 | 글로벌 시장 개발자 도구 | 성공 확률 40-50% |

**실패 비용은 0원이 아니라 최소 327만원(기회비용 하한선)이다.**

### 도전 2: 역방향 검색은 검증되지 않은 가설

- 실제 학부모/학생에게 **한 번도 물어보지 않았다**
- 맘카페 수요는 "비교"가 아니라 "정보" (단일 대학 질문이 대부분)
- 역방향 검색으로 50개 학과 목록을 받아도, 결국 각 대학을 다시 클릭해봐야 함
- **"있으면 좋은(Vitamin)" ≠ "없으면 안 되는(Painkiller)"**

### 도전 3: 기대값 계산

| 시나리오 | 확률 | 가치 | 기대값 기여 |

**같은 시간 투자로 기대값 차이가 약 290만원.**

### 도전 5: 가장 잔인한 질문

> **"이 프로젝트가 없는 세상에서, 누가 진짜 불편해하는가?"**

솔직한 답: **아무도 진짜 불편해하지 않는다.** 대입정보포털, 대학 홈페이지, 맘카페, 입시 컨설팅, 학교 상담 — 5개 경로가 이미 존재한다.

### 그럼에도 불구하고 Go를 지지하는 유효한 논거

1. **학습 가치는 실재**: Next.js + SEO + 실사용자 대상 서비스 경험의 조합은 희소
2. **"더 좋은 대안" 함정**: 더 좋은 아이디어를 찾다가 아무것도 안 하는 것이 가장 나쁜 결과
3. **109시간은 인생에서 크지 않다**: 연간 깨어있는 5,840시간 중 1.9%
4. **이미 분석 완료**: 5라운드 분석 자체가 매몰되지 않은 가치

### 악마의 변호인이 제시하는 Go의 전제 조건

| 조건 | 기한 | 불충족 시 |
|------|------|----------|

| **학습 목적을 1차 목표로 설정**: "사용자 확보"가 아니라 "학습" | 즉시 | 자기기만 |
| **4주 체크포인트**: MVP 배포 가능 상태 | 4주차 | **중단** |
| **대안 프로젝트 리스트 작성**: 중단 시 즉시 전환할 대안 1-2개 | 즉시 | - |
