### 공격 2: 2028 교육과정 데이터 시간축 모순 (가장 치명적)

> 현재 데이터: 2015 개정 교육과정 (물리학I, 물리학II, 화학I...)
> 2028 고교학점제: 2022 개정 교육과정 (역학과 에너지, 전자기와 빛, 물질과 에너지...)
> **과목 체계가 완전히 다르다.**

| 2015 개정 (현재 데이터) | 2022 개정 (2028 기준) |
|----------------------|---------------------|
| 물리학I, 물리학II | 물리학, 역학과 에너지, 전자기와 빛 |
| 화학I, 화학II | 화학, 물질과 에너지, 반응 속도와 촉매 |
| 수학I, 수학II | 대수, 미적분I, 미적분II |

**지금 만드는 정규화 사전과 데이터는 2028년에 전면 폐기될 수 있다.** 대학의 2028 시행계획은 2027년 4-5월 발표 예정.

**발생 확률: 95% | 영향도: 치명적**

### 공격 3: SEO "블루오션"의 허상

- "2028 대입 권장과목 비교" 같은 키워드는 **수요가 없어서 경쟁이 없는 것**일 가능성
- 검색 수요 본격화는 **2027년 하반기** → 1년간 트래픽 0 인내 필요
- 네이버에서 외부 사이트 노출 극히 제한적 (교육 검색의 65%가 네이버)
- 500개 정적 페이지 → Google **thin content 페널티** 위험 (각 페이지에 고유 콘텐츠 부재)

### 공격 4: 재방문율 구조적 한계

과목 선택은 **1-2회성 의사결정** (고1→고2, 고2→고3). 서비스를 10번 방문할 사용자는 없다. DAU/MAU 비율이 극히 낮을 수밖에 없는 **구조적 한계**.

### 공격 5: 경쟁자 대응

## 2. Blue Team (방어자): "약점은 인정하되, 실패 비용은 0원"

### 방어 1: 타임라인은 범위 조정으로 해결 가능

- 면책조항, AI 제거, 문구 수정은 **30분 이내** 완료 가능 (코드 1-2줄 변경)
- 과목 정규화 사전: 30-40개 엔트리면 충분. 변이 패턴이 제한적 (공백/로마자/숫자 차이)
- **대안**: PDF 구조화를 2단계로 분리
  - 1단계 (Week 1-2): 상위 5개 대학만 완전 구조화
  - 2단계 (Week 3-4와 병행): 나머지 10개 추가

### 방어 2: JSON+SSG는 이 규모에 최적

| 항목 | 수치 |
|------|------|
| 전체 데이터 포인트 | 15대학 x 30학과 = 450개 |
| JSON 크기 | 200-300KB |
| 빌드 시간 | 800페이지 x 100ms = 80초 |
| Vercel 빌드 제한 | 45분 (무료) |

DB는 **오버엔지니어링**. Git 버전 관리, 빌드 타임 검증(`zod`), CDN 자동 배포가 장점.

### 방어 3: 리텐션 프레임이 틀렸다

> **"같은 사용자의 재방문"이 아니라 "매년 새로운 코호트 진입"이 성장 모델이다.**

- 매년 약 50만 고교 신입생 + 학부모 195만 = **매년 100만명 이상의 잠재 사용자 갱신**
- 수능 점수 계산기도 연 1회 사용이지만 매년 50만 수험생이 새로 유입
- 핵심은 리텐션이 아니라 **연간 반복 유입**

### 방어 4: 경쟁자가 안 하는 구조적 이유

1. **비즈니스 모델 충돌**: 진학사 수익원은 고3 합격예측 → 고1 과목선택은 수익 기여 낮아 우선순위 밀림
2. **정규화 데이터의 보이지 않는 비용**: "확률과 통계" vs "확률과통계" 매핑은 도메인 지식 필요
3. **경쟁자 복제 = 시장 검증**: 따라온다면 시장이 존재한다는 증거

### 방어 5: 실패 비용이 0원

- 투자: Vercel 무료 + 본인 시간
- 실패 시 손실: 시간 (그러나 이것 자체가 Next.js/SSG/SEO 실전 포트폴리오)
- 성공 보상: 트래픽 + 수익화 + 포트폴리오

### Plan B 매트릭스

| 실패 시나리오 | 감지 시점 | 감지 지표 | 대응 전략 |
|-------------|---------|----------|----------|
| SEO 실패 | Week 8-10 | Search Console 노출 < 100/주 | 맘카페 직접 유입, 교사 네트워크 |
| 데이터 번아웃 | Week 3-4 | 주 4시간+ 데이터 작업 | 5개 대학으로 축소, 커뮤니티 기여 모델 |
| 경쟁자 동일 기능 출시 | 지속 모니터 | ipsihogu/진학사에 유사 기능 | 니치 깊이 방어, B2B 피봇 |

## 3. Market Analyst (시장 분석가): "데드라인은 4월 말, 첫 코호트는 이미 놓쳤다"

1. **"역방향 검색이 핵심 차별화"** → 검증 필요: 맘카페 설문 또는 랜딩페이지 A/B 테스트

3. **"SEO로 MAU 5,000-15,000 달성"** → Google Trends/네이버 데이터랩에서 실제 검색량 확인 필요

## 7. 4라운드 핵심 결론

### 가장 위험한 3가지 (3라운드에서 간과된 것)

1. **2022 개정 교육과정 과목 체계 전환** — 현재 데이터가 2028에 무효화될 가능성. 3라운드에서 **아무도 언급하지 않았던** 치명적 리스크
2. **타임라인의 물리적 불가능** — Week 1-4에 80시간 분량이 33.6시간에 압축. 계획 수정 없이는 3주차에 좌절
3. **첫 코호트 이탈** — 2028 대입 대상자(현 고2)의 과목선택은 이미 2025년에 종료. 타겟 재정의 필요

### 수정된 실행 계획 (4라운드 합의)

```
Phase 1 — "4월 말 MVP" (8주, ~67시간)
  Week 1-2: 법적 정비 (3h) + 과목 정규화 사전 (4h) + JSON 분리 (3h) + 기존 6개 대학 검증 (5h)
  Week 3-4: 3개 대학 추가 (9h) + SSG 전환 (8h) + Search Console/네이버 등록 (2h)
  Week 5-6: 역인덱스 + 역방향 검색 UI (13h)
  Week 7-8: 학과별 정적 페이지 ~200개 (8h) + OG/공유 (8h) + 배포 (4h)

  → 5월 과목선택 시즌에 9개 대학, 역방향 검색 포함 MVP 공개

Phase 2 — "데이터 확장 + SEO" (5주, ~42시간)
  Week 9-10: 나머지 6개 대학 추가 (12h) + 인문계열 데이터 (9h)
  Week 11-12: 블로그 3편 (6h) + 랭킹 페이지 (7h) + 추가 정적 페이지 (5h)
  Week 13: 맘카페 침투 시작 (3h)

Phase 3 — "2028 대비" (별도 일정)
  2026.05: 2022 개정교육과정 과목 체계 매핑 조사
  2027.04-05: 2028 시행계획 발표 시 즉시 데이터 전환
```
