> 정리일: 2026-03-03
> 범위: Agent Council 8라운드 (32명 AI 에이전트) + 국책사업 조사 + 현실성 검증 + 맘카페 분석 + 피봇 전략

**"2028 고교학점제에 대비하여, 대학별 권장과목을 비교하고 역방향 검색(내 과목 → 가능한 대학)할 수 있는 무료 웹서비스"**

## I. 프로젝트 현재 상태

### 기술 스택
- Next.js 16 + React 19 + Tailwind 4 + TypeScript
- 5개 소스 파일, ~620줄
- CSR 상태 (`"use client"`) — SSG 미전환

### 보유 데이터
| 자산 | 상태 | 위치 |
|------|------|------|
| 6개 대학 권장과목 (하드코딩) | 동작, 웹에 연결 | `web/src/lib/course-data.ts` (249줄) |
| 6개 대학 권장과목 (JSON) | 중복, 웹 미연결 | `data/course-guide/raw_recommendations.json` |
| 15개 대학 입시정보 JSON | **풍부** (전형/수능최저/모집인원/교과이수) — 단, 권장과목 없음 | `data/universities/*.json` |
| 대기업 연계학과 18개 | 잘 구조화됨, 웹 미연결 | `data/corporate-tracks/all-tracks.json` |
| 대학 목록 메타데이터 | 인서울15 + 의대38 + 계약학과 | `data/meta/target-universities.json` |
| JSON 검증 스키마 2개 | 정의 완료 | `data/schemas/university.schema.json`, `corporate-track.schema.json` |
| 의대 데이터 | **비어있음** (디렉토리만 존재) | `data/medical-schools/` |
| 맘카페 34편 시리즈 | 분석 완료 | `docs/맘카페 엄마의 실제 사례/` |
| 네이버 카페 크롤러 | COOKIE 미설정, 미실행 상태 | `naver_cafe_crawler.py` |

### 구현된 기능
- 대학 간 권장과목 비교 (6개 대학)
- 기본 랜딩 페이지 + 가이드 페이지

### 미구현 핵심 기능
- 역방향 검색 (과목 → 대학)
- SSG 전환 (SEO의 전제조건)
- 데이터 확장 (6개 → 45개 대학)

## II. 시장 분석 요약

### 수요 증거
| 출처 | 증거 | 강도 |
|------|------|------|
| 맘카페 34편 시리즈 | 수만명 구독, 32탄이 권장과목 주제 | 강함 |
| 고교학점제 전면 시행 (2026.3) | 과목 선택 도구 필요성 최고조 | 강함 |
| adiga.kr 45개 대학 PDF | 대교협이 데이터를 모았다 = 수요 인정 | 중간 |

### 시장 규모
- TAM: 325만명 (고등학생 130만 + 학부모 195만)
- SAM: 78만명 (온라인 검색 60% × 과목선택 관심 40%)
- 매년 새로운 코호트 100만명+ 유입

### 경쟁 환경
| 경쟁자 | 강점 | 약점 |
|--------|------|------|
| adiga.kr (대교협) | 공식, 45개 대학, 무료 | PDF만 제공, 비교 불가 |
| ipsihogu.com | 검색 기능, 정리됨 | 유료 |
| 진학사 | 풍부한 입시 데이터 | 과목 비교 기능 없음 |
| 맘카페 | 커뮤니티 신뢰 | 정보 산발적 |

### 유일한 차별점
> **"A대학 vs B대학 vs C대학의 권장과목을 한 화면에서 비교"하는 서비스는 시장에 존재하지 않음** (1라운드 Researcher 발견, 8라운드까지 반박 없음)

### 핵심 데이터 공백 (= 사업 기회)
- **대학별 권장과목**: 공개 API 없음 → 각 대학 PDF 개별 파싱 필요
- **어디가(adiga.kr) 데이터**: API 미공개, 웹 조회만 가능
- 이 공백을 메우는 것이 서비스의 핵심 가치

## III. 8라운드 Agent Council 핵심 발견

### 라운드별 진화
```
1라운드 — "기회가 있다!" → 횡단 비교가 시장 공백
2라운드 — "방향을 잡았다!" → 역방향 검색이 핵심, SEO가 전쟁터
3라운드 — "계획도 세웠다!" → 13주 실행 계획, adiga.kr에 2028 데이터 존재 발견
4라운드 — "시간이 안 된다" → 13주 불가능, 8주 67시간으로 축소, 성공률 20-30%
5라운드 — "기대값이 마이너스" → -68.5만원, Kill Switch 필요
6라운드 — "분석 자체가 편향" → AI 20명의 구조적 편향, 사용자 검증 0회
7라운드 — "2인+CC+국책사업 = 게임 체인저" → 기대값 +500~1,800만원 (국책사업 포함 시)
8라운드 — "기능보다 데이터" → 데이터 확장 + SEO > 새 기능 10개
```

### 32명 전원 합의 사항
1. **역방향 검색은 시장에 없는 기능이다**
2. **SSG 전환이 모든 것의 전제조건이다**
3. **데이터 출처(ipsihogu.com) 즉시 교체 필수**
4. **사용자 검증이 코드 한 줄보다 먼저다**
5. **국책사업이 기대값을 뒤집는 유일한 변수다**
6. **기능 추가보다 데이터 커버리지 확장이 3-10배 높은 ROI**

### 성공 확률 변화
```
1라운드  40-50% (암묵적)
4라운드  20-30% (스트레스 테스트 후 급락)
7라운드  2인+CC 단독: 25-35% / 2인+CC+국책사업: 55-65%
8라운드  데이터+SEO 경로: 50-65% / 기능 추가 경로: 25-38%
```

### 기대값 비교
| 시나리오 | 기대값 |
|---------|--------|
| 현재 프로젝트 (1인, 기존 계획) | **-68.5만원** |
| 2인 + Claude Code (국책사업 없이) | **-130만원** (악화) |
| 2인 + Claude Code + 국책사업 | **+500~1,800만원** (반전) |
| 대안 프로젝트 (개발자 도구 비교) | **+197.8만원** |

### 해결하지 못한 근본 질문
1. 역방향 검색을 원하는 사용자가 실재하는가? (검증 0회)
2. 이 서비스가 없으면 누가 진짜 불편한가?
3. 2년 후 2022 개정 교육과정 전환 시 다시 만들 의지가 있는가?

## IV. 맘카페 분석 핵심

### 34편 시리즈에서 발견한 것
- **학부모는 "정방향"으로 사고**: 목표 대학 → 필요한 과목 (역방향 아님)
- **비교가 최대 Pain Point**: "A대학과 B대학 권장과목이 뭐가 다른지" 비교할 도구 없음
- **80%가 비교 단계에서 포기** → EntranceExamPlanner가 해결하는 지점
- **32탄이 직접 관련**: "2028학년도 권장과목 정리" — "실질적 영향력은 매우 클 것"

### 학부모 의사결정 여정
```
불안감(100%) → 정보수집(80%) → 비교(20%) → 전문가상담(5%)
                                  ↑ 80% 이탈 지점
```

### 시리즈 작성자 = 이상적인 첫 사용자/앰버서더
- 수만명의 팔로워 보유
- 32탄에서 권장과목 주제를 직접 다룸
- "이 글에서 말한 비교를 한 번에 할 수 있는 도구"로 연결 가능

## IX. 8라운드 누적 인사이트: "분석은 끝났다"

### 8라운드의 패턴
```
매 라운드 "이것만 하면" 패턴 반복:
1라운드 "SSG로 전환하면" → 2라운드 "역방향 검색 추가하면" →
3라운드 "정규화하면" → 4라운드 "표기 통일하면" →
5라운드 "피봇하면" → 6라운드 "정직해지면" →
7라운드 "두 명이면" → 8라운드 "기능 추가하면"

실행: 0회
```

### 가장 비용 효과적 액션 (전원 합의)
| 순서 | 액션 | 시간 | 효과 |
|------|------|------|------|
| **1** | 데이터 출처 표기 수정 | 10분 | 리스크 제거 |
| **2** | 현재 버전 맘카페 공유 | 30분 | +15-20% 성공 확률 |
| **3** | 학부모 3명 인터뷰 | 3시간 | 방향 검증 |
| **4** | SSG 전환 | 2시간 | SEO 기반 구축 |
| **5** | 45개 대학 데이터 확보 계획 | 1시간 | 확장 로드맵 |

### 성공 공식 (8라운드 종합)
```
성공 = 데이터 커버리지(45개 대학) × SEO(SSG + 900페이지) × 사용자 검증
     > 기능 추가 10개
```
