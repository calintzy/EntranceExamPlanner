## VI. 24명의 에이전트가 끝내 해결하지 못한 질문들

6라운드를 거쳐도 답이 나오지 않은 근본 질문들입니다.

### 1. "역방향 검색을 원하는 사용자가 실재하는가?"
- 24명의 AI가 "이것이 핵심"이라 합의했지만, **단 1명의 학부모도 확인하지 않음**
- 맘카페 34편 분석은 "단일 대학 질문"이 대부분. "내 과목으로 어디 갈 수 있나?"는 추론

### 2. "이 서비스가 없으면 누가 진짜 불편한가?"
- 5라운드 Devil's Advocate의 가장 잔인한 질문
- 솔직한 답: 대입정보포털, 대학 홈페이지, 맘카페, 입시 컨설팅, 학교 상담 — **5개 대안이 이미 존재**

### 3. "개발자의 시간을 가장 가치 있게 쓰는 방법은 이 프로젝트인가?"
- 6라운드 Pivot Strategist가 제시한 대안(개발자 도구 비교)의 기대값이 **+266만원 더 높음**
- 그러나 "기대값이 높은 프로젝트"와 "하고 싶은 프로젝트"는 다른 문제

### 4. "2년 후 2022 개정 교육과정 전환 시 이 서비스를 다시 만들 의지가 있는가?"
- 4라운드 Red Team이 발견한 근본 리스크
- 현재 데이터의 수명은 최대 2년. 그 이후는 사실상 새 프로젝트

## VII. AI 분석의 한계: 이 총평 자체에 대한 메타 비판

### 이 분석이 할 수 있었던 것
- 코드베이스의 기술적 상태를 정확히 진단
- 시장 구조와 경쟁 환경을 논리적으로 분석
- 실행 계획의 물리적 실현 가능성을 수치로 검증
- 법적 리스크를 식별하고 대응 방안 제시
- 대안 프로젝트의 기대값을 비교

### 이 분석이 할 수 없었던 것
- **실제 사용자의 반응을 예측** — 학부모 5명에게 보여주는 것이 6라운드 분석보다 가치 있음
- **개발자의 내적 동기를 판단** — "하고 싶다"는 감정은 기대값 계산으로 대체할 수 없음
- **미래의 경쟁 환경을 확정** — 진학사가 내일 역방향 검색을 출시할 수도, 10년간 안 할 수도 있음
- **"학습의 가치"를 정량화** — 이 프로젝트를 통한 학습과 다른 프로젝트를 통한 학습의 비교는 불가능

### 24명 AI의 구조적 편향 (6라운드 Self-Deception Detector)

| 편향 | 설명 |
|------|------|
| 긍정적 결론 편향 | AI는 "하지 마"라고 말하도록 훈련되지 않음 |
| 역할 비대칭 | Go 방향 역할이 항상 2-3배 많았음 |
| 누적 합의 압력 | 3라운드부터 "할 것인가"가 아닌 "어떻게 할 것인가"로 프레임 고착 |
| 분석 매몰비용 | 5라운드까지 분석한 것 자체가 Go 방향 편향 유발 |

## VIII. 최종 판정: 세 가지 길

6라운드의 분석을 종합하면, 앞으로 갈 수 있는 길은 세 가지입니다.

### 길 A: "5라운드 계획대로 실행"
```
8주 67시간 투입 → 9개 대학 + 역방향 검색 + 200페이지 SSG
성공 확률: 20-30%
기대값: -68.5만원
장점: 완결성 있는 MVP
위험: 기대값 마이너스인 프로젝트에 시간을 추가 투입
```

### 길 B: "즉시 배포 + 피봇" (6라운드 권장)
```
현재 상태(6개 대학 비교) 그대로 Vercel 배포 → SEO 시계 시작
남은 시간 70%를 기대값이 더 높은 프로젝트에 투자
성공 확률 (둘 중 하나): 40-50%
기대값: +80~120만원
장점: 리스크 분산, 기대값 극대화
위험: 두 프로젝트 모두 반쪽짜리 가능
```

### 길 C: "학부모 5명에게 먼저 물어보고 결정"
```
이번 주: 맘카페에서 학부모 5명에게 역방향 검색 콘셉트 검증
3명+ 긍정 → 길 A 실행 (사용자 검증된 Go)
2명 이하 → 길 B 실행 (피봇)
소요: 2시간
가치: 6라운드 24명 AI 분석보다 높을 수 있음
```

## IX. 이 분석에서 진짜로 배운 것

기술적 발견, 시장 분석, 실행 계획을 넘어서 — 6라운드를 통해 드러난 **프로젝트 의사결정에 관한 메타 교훈**입니다.

### 1. 분석은 결정을 대신하지 않는다
5라운드의 정교한 분석이 오히려 단순한 진실을 가렸습니다. "하고 싶다"는 충분한 이유이고, "기대값이 마이너스다"는 중요한 정보입니다. 둘 다 인정하면 됩니다.

### 2. 성공 확률이 낮아질수록 목표를 재정의하는 것은 자연스럽지만 위험하다
"수익화 가능한 서비스" → "학습 투자"로의 전환은 합리적으로 보이지만, 실패를 인식할 수 없게 만드는 부작용이 있습니다.

### 3. 가장 가치 있는 검증은 가장 저렴한 검증이다
24명의 AI 에이전트 × 6라운드 < 학부모 5명 × 30분 대화. 코드 한 줄 짜기 전에 사용자에게 물어보는 것이 가장 높은 ROI.

### 4. Kill Switch는 외부 강제력 없이는 작동하지 않는다
자기 자신에게 설정한 마감은 자기 자신이 연장합니다. Kill Switch가 진짜 작동하려면 제3자에게 공개 선언해야 합니다.

### 5. "실패 비용 0원"은 모든 프로젝트에 적용되므로 의사결정 기준이 아니다
진짜 질문은 "이 시간을 어디에 쓰는 것이 가장 가치 있는가"입니다.
