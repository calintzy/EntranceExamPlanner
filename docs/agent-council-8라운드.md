# Agent Council 8라운드: "기능을 추가하면 성공 확률이 올라갈까?"

> 분석일: 2026-03-03
> 참여 에이전트: 기능 전략가, 학부모 대변인, 기능 회의론자, 그로스 해커
> 주제: 학부모 수요가 확인된 상황에서, 기능 추가가 성공 확률을 높이는가?

---

> **4명 전원이 "기능 추가 자체는 성공 확률을 높이지 않는다"에 합의했습니다.**
>

>
> 그리고 가장 비용 대비 효과가 큰 액션은 "지금 당장 맘카페에 현재 버전을 공유하는 것"(30분, +15-20%)입니다.

## 1. 기능 전략가: "데이터 확장이 기능 추가를 이긴다"

### 기능별 성공 확률 기여도

| 우선순위 | 기능 | 성공 확률 기여 | 난이도 | ROI |
|---------|------|-------------|--------|-----|
| **#1** | **SSG 전환 (SEO)** | **+25-30%** | 낮음 | **최고** |
| **#2** | **데이터 확장 6→15개 대학** | **+15-20%** | 중간 | **높음** |
| **#3** | **역방향 검색** | **+5-10%** | 낮음 | 중간 |
| #4 | 과목 체크리스트 | +2-3% | 낮음 | 낮음 |
| #5 | 비교 결과 공유(딥링크) | +1-2% | 낮음 | 낮음 |

### MFS (Minimum Feature Set) = 45-60% 성공률

```
현재: 20-30% 성공률

MFS = SSG 전환 + 15개 대학 데이터 + 역방향 검색
    = 20-30% + 25-30% + 15-20% + 5-10%
    = 45-60% 성공률 (중복 보정 후)
```

### "450+ SEO 페이지" 전략

- 15개 대학 × 30개+ 학과 = **450개 이상의 정적 페이지**
- 각 페이지가 독립적인 SEO 랜딩 페이지로 작동
- "서울대 컴퓨터공학과 권장과목" 같은 롱테일 키워드 독점 가능
- 현재 이 키워드들에 대한 구조화된 HTML 경쟁자 = **거의 0**

### 학부모 Pain Point별 실현 가능성

| Pain Point | 내용 | 구현 가능? |
|-----------|------|----------|
| #1 권장과목 비교 | 대학 간 비교 | **이미 구현됨** |
| #2 합격선 정보 | 입시 결과 데이터 | ❌ 불가능 (데이터 없음) |
| #3 수능 전략 | 수능 과목 선택 | ❌ 범위 밖 |
| #4 교과전형 변화 | 전형 변화 추적 | △ 부분 가능 |
| #5 권장과목 의미 | 영향도 해석 | △ 콘텐츠로 가능 |
| **#6 학교별 개설과목** | 우리 학교에 있는지 | ❌ 데이터 없음 |

**결론: 구현 가능한 Pain Point는 #1뿐 — 이미 핵심 기능으로 보유 중**

### 핵심 메시지

> **"새 기능 10개보다 기존 기능의 데이터 커버리지를 6→45개 대학으로 확장하는 것이 성공 확률을 3배 더 올린다. 기능이 아니라 데이터가 이 서비스의 해자(moat)다."**

## 2. 학부모 대변인: "새 기능보다 데이터 신뢰가 먼저"

### 학부모의 의사결정 여정

```
[불안감 발생] → [정보 수집] → [비교] → [전문가 상담]
    100%          80%         20%        5%
                              ↑
                        80%가 여기서 포기
                    (비교 도구 부재가 원인)
```

**EntranceExamPlanner는 정확히 이 "비교" 단계의 이탈을 막는 도구**

### 현재 서비스의 결정적 강점

- **원클릭 대학 간 비교** — 시장에 유일
- 학부모가 가장 많이 쓰는 행위: "A대학 B학과 vs C대학 D학과, 권장과목 뭐가 다른지"
- 이 기능 하나가 adiga.kr PDF 다운로드보다 10배 편리

### 가장 심각한 문제: 신뢰

| 문제 | 현재 상태 | 심각도 |
|------|----------|--------|
| **데이터 출처 불일치** | `course-data.ts:2` "ipsihogu.com" vs `page.tsx:111` "공식 모집요강" | **치명적** |
| 업데이트 날짜 없음 | 데이터가 언제 기준인지 불명 | 높음 |
| 검증 마크 없음 | "이 정보가 맞는지" 확인 불가 | 높음 |
| 약관/면책 조항 부실 | 법적 보호 미흡 | 중간 |

**학부모 관점: "이 사이트가 맞는 정보를 주는 건지 모르겠다" = 즉시 이탈**

### 리텐션을 위한 킬러 기능: "내 아이 과목 체크리스트"

```
[내 아이가 선택한 과목 입력]
    → [목표 대학 권장과목과 매칭]
    → [부족한 과목 알림]
    → [저장/공유 가능]
```

- 1회성 방문 → 반복 방문으로 전환
- 학부모 간 공유 시 바이럴 루프 형성
- **그러나 이것도 데이터 커버리지가 15개 대학 이상일 때만 의미**

### 핵심 메시지

> **"새 기능 10개보다, 데이터 정확성에 대한 확신 1개가 더 큰 가치입니다. '이 정보를 믿어도 되는가?'가 학부모의 첫 번째 질문이고, 현재 서비스는 이에 답하지 못하고 있습니다."**

### "수요가 있다" ≠ "우리 서비스를 쓸 것이다"

학부모들이 사용하는 **이미 존재하는 대안들:**

| 대안 | 장점 | 단점 |
|------|------|------|
| **adiga.kr** | 대교협 공식, 45개 대학, 무료 | PDF, 비교 불가 |
| **ipsihogu.com** | 검색 기능, 정리됨 | 유료 |
| **맘카페** | 커뮤니티 신뢰, 실시간 | 정보 산발적 |
| **대학 홈페이지** | 공식 원본 | 각각 방문 필요 |

**EntranceExamPlanner가 이겨야 하는 상대:** 무료 + 공식 + 45개 대학(adiga.kr)

- 현재 6개 대학에서도 데이터 불일치 문제 존재
- 확장 시 법적 리스크 비례 증가 (모집요강 저작권)

### 가장 비용 효과적인 액션 TOP 3

## 4. 그로스 해커: "기능 < 데이터 < SEO — 성장 공식은 정해져 있다"

### 성장 공식

```
트래픽 = SEO(70%) + 맘카페 바이럴(20%) + 학교 채널(10%)
```

### ROI 순서: 데이터 > 기능 > 마케팅

| 투자 | ROI | 이유 |
|------|-----|------|
| **데이터 확장 (6→45개 대학)** | **10x** | 900+ SEO 페이지, 경쟁 0 |
| 기능 추가 (역방향 검색) | 3x | 45개 대학 데이터 있을 때만 의미 |
| 마케팅 (맘카페 공유) | 2x | 데이터 부족하면 이탈 |

### SEO 무기: 900+ 정적 페이지

```
45개 대학 × 20개+ 학과 = 900+ 고유 페이지

각 페이지 = 1개의 SEO 랜딩 포인트:
  /대학/서울대학교/컴퓨터공학부
  /대학/연세대학교/전기전자공학부
  /과목/미적분II → 이 과목을 권장하는 대학 목록
  /비교/서울대-연세대/컴퓨터공학

→ 현재 이 키워드들의 구조화된 HTML 경쟁 = 거의 0
→ Google "서울대 컴퓨터공학 권장과목" 검색 시 1페이지 독점 가능
```

### 맘카페 바이럴 전략

1. **34편 시리즈 작성자를 첫 번째 사용자로** — 이미 수만명의 팔로워 보유
2. 시리즈 32탄(2028 권장과목)과 직접 연결 — "이 글에서 말한 비교를 한 번에 할 수 있는 도구"
3. 학부모 간 비교 결과 공유 → 자연 바이럴 루프

### 3단계 로드맵

#### Phase 1: SEO 착수 (0-6주)
| 작업 | 효과 |
|------|------|
| SSG 전환 | SEO 기반 구축 |
| 45개 대학 데이터 확보 (adiga.kr 기반) | 900+ 페이지 |
| sitemap + JSON-LD | 검색엔진 인덱싱 |
| 역방향 검색 구현 | 차별화 포인트 |

#### Phase 2: 트래픽 확보 (6주-6개월)
| 작업 | 효과 |
|------|------|
| 맘카페 바이럴 캠페인 | 초기 사용자 확보 |
| Search Console 최적화 | 오가닉 트래픽 |
| 콘텐츠 마케팅 (입시 가이드) | 체류 시간 증가 |

#### Phase 3: 수익화 (6-18개월)
| 작업 | 효과 |
|------|------|
| 프리미엄 기능 (상세 분석) | B2C 수익 |
| 학원/학교 대시보드 | B2B 수익 |
| 국책사업 자금 | 비대칭 수익 |

### Kill Switch

```
Phase 1 완료 후 3개월 시점:
  월간 UV < 2,000 → 피봇
  월간 UV 2,000-10,000 → 유지 + 최적화
  월간 UV > 10,000 → 확장
```

### 핵심 메시지

> **"기능을 추가할까? 라는 질문 자체가 잘못되었습니다. 이 서비스의 성장 엔진은 '기능'이 아니라 '데이터 커버리지 × SEO 페이지 수'입니다. 45개 대학 데이터 = 900개 SEO 페이지 = 경쟁자 0인 블루오션. 기능은 트래픽이 온 후에 추가해도 늦지 않습니다."**
