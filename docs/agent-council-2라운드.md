# Agent Council 2라운드: 방향성 토론

> 분석일: 2026-03-01
> 참여 에이전트: Builder (옹호자), Skeptic (회의론자), Strategist (전략가), User Advocate (사용자 대변인)
> 분석 대상: 1라운드 분석 결과를 바탕으로 한 "그래서 어떻게 할 것인가?" 토론

## 토론 구조

1라운드에서 "가능성은 있지만 현재 방향 그대로는 안 된다"는 결론이 나왔습니다.
2라운드에서는 4명의 새로운 역할로 **구체적 방향성**을 토론했습니다.

## 1. Builder (옹호자): "지금 시작하지 않으면 부적격이 되는 프로젝트"

### Critic의 "투자 부적격" 판정 반박

| Critic 리스크 | Builder 반박 | 인정 여부 |
|---|---|---|
| 68시간/년 수작업 | 주 1.3시간일 뿐. AI 파이프라인 적용 시 10시간/년으로 축소 가능 | 부분 타당 |
| 법적 리스크 18억 | 허위 광고 사업자에 대한 과징금. 무료 참고도구에 동일 적용은 논리 비약 | 타당 |
| 2-3주 내 복제 | 기존 플레이어가 안 하는 이유 존재 (자사 비즈니스 모델과 충돌) | 논쟁적 |
| 수익화 벽 | 1인 사이드 프로젝트에 VC 기준 적용이 잘못된 프레임 | 타당 |
| 하드코딩 | MVP의 정상적 상태. 1-2주 리팩토링으로 해결 | 타당 |
| 데이터 시의성 | 2026은 연습, 2028이 본게임. 오히려 기회 | 부분 타당 |
| 1인 번아웃 | 범위 한정 + AI 자동화 + 커뮤니티 기여로 관리 가능 | 부분 타당 |

### "해자(moat)" 구축 전략 4가지

1. **역방향 검색 알고리즘 선점** — "내 과목 → 지원 가능 대학" 기능을 시장에서 가장 먼저 제공
2. **과목명 정규화 데이터 축적** — 대학마다 다른 표기("확률과 통계" vs "확률과통계")의 매핑 사전
3. **맘카페 인플루언서 네트워크** — 기술로 복제 불가능한 관계 자산
4. **2028 데이터 선점** — 고교학점제 전면 적용 첫 해 데이터를 가장 먼저 구조화

### 90일 실행 계획 (주 8.4시간, 총 109시간)

| Phase | 기간 | 핵심 작업 |
|-------|------|----------|
| 1 (기반) | 1-3주 | 데이터-코드 분리, 역방향 검색 구현, 과목명 정규화 |
| 2 (차별화) | 4-7주 | 횡단 비교 UI 고도화, SEO, AI PDF 파싱, 맘카페 홍보 |
| 3 (성장) | 8-13주 | 피드백 반영, 2028 데이터, 시뮬레이터, B2B 시범, 광고 |

### 핵심 논거: "잘못된 프레임으로 정확한 평가는 불가능"

> 1인 사이드 프로젝트에 "투자 부적격"이라는 판정을 내리는 것은, 자전거 출퇴근자에게 "이 차량은 고속도로 주행 부적격"이라고 말하는 것과 같다.

투자: Vercel 무료 + 본인 시간. 실패 비용: 0원. 성공 보상: 포트폴리오 + 트래픽 + 수익화 가능성.

## 2. Skeptic (회의론자): "아무도 안 하는 데는 이유가 있다"

### "횡단 비교" 차별화에 대한 3가지 도전

**첫째, 실제 수요 증거가 없다.**
맘카페 34편은 "비교해주세요"라는 요청이 아니라 "우리 아이 목표 대학에 맞는 과목이 뭔지" 알려달라는 단일 대학 질문. 횡단 비교와 단일 대학 조회는 완전히 다른 니즈.

**둘째, 대학 간 권장과목 차이가 크지 않다.**
컴퓨터공학은 어디나 미적분/기하/확률과통계가 핵심. 의예과는 어디나 화학/생명과학이 핵심. "비교할 만한 차이"가 실제로 별로 없을 수 있음.

**셋째, 기존 경쟁자가 안 만드는 이유가 "수요 부족"일 가능성.**
Builder는 "비즈니스 모델 충돌"이라 했지만, 진학사가 횡단 비교를 추가하는 것과 비즈니스 모델이 충돌할 이유가 전혀 없음.

### Builder 90일 계획의 허점

| 문제 | 내용 |
|------|------|
| **데이터 수집 노동량 과소평가** | 300개 학과 x 15분/학과 = 75시간. 총 예산 109시간의 75% 소진 |
| **학과명 매칭 복잡성** | "기계공학부" vs "기계공학과", "전기전자공학부" vs "전기·정보공학부" 정규화 작업만 수십 시간 |
| **데이터 출처 법적 문제** | `course-data.ts`에 "출처: ipsihogu.com"이지만, UI에는 "출처: 각 대학 입학처"로 표시. 불일치 |

### 놓친 리스크 4가지

| 리스크 | 내용 |
|--------|------|
| **데이터 출처 법적/윤리적 문제** | ipsihogu.com 데이터를 무단 사용하면서 "대학 입학처 출처"로 표기하는 것은 기만적 |
| **데이터 정확성 책임** | 오정보로 과목 선택한 학생이 SNS에 불만 퍼뜨리면 평판 즉시 붕괴 |
| **인문계 커버리지 부재** | 현재 자연계열만 커버. 전체 입학생의 ~40%가 인문계열 |
| **권장과목의 실질적 영향력** | "권장"이지 "필수"가 아님. 합불을 결정하는 핵심 변수가 아닐 수 있음 |

### 핵심 질문

> ipsihogu.com이 이미 존재하는데, "횡단 비교" 때문에 사용자가 이 서비스를 선택할 구체적 증거는 무엇인가?
>
> "아무도 안 하는 것"과 "시장 기회"를 혼동한 것은 아닌가?

## 3. Strategist (전략가): "교과 설계 도구라는 새 카테고리를 SEO로 선점하라"

### 전략 선택: A+B 복합 전략 (순서가 핵심)

```
Phase 1: A(무료 도구)로 트래픽 확보 + SEO 자산 구축
Phase 2: 트래픽 기반으로 광고 수익 + 프리미엄 기능 테스트
Phase 3: 사용 데이터를 근거로 B(B2B) 영업 시도
```

C(AI 챗봇)는 보류: 입시에서 할루시네이션은 치명적
D(커뮤니티)는 비추: 냉시동 문제 해결 불가

### GTM: 첫 1,000명 확보 전략

| 채널 | 비용 | 예상 효과 | 우선순위 |
|------|------|----------|----------|
| **네이버 SEO** | 0원 | 3-6개월 후 월 5,000-15,000 방문자 | 최우선 |
| **맘카페 오가닉 침투** | 0원 (시간 투자) | 월 500-1,000 (느리지만 충성도 높음) | 2순위 |
| **역방향 검색 바이럴** | 0원 | 바이럴 시 단기 1,000-5,000 | 3순위 |
| **교사 네트워크** | 0원 | 교사 1명 = 학년 300-400명 추천 | 중기 |

핵심 실행: SSG 전환으로 170개 학과 x 6개 대학 = **1,020개 정적 URL 생성** → 롱테일 SEO 공략

### 수익화 타임라인

| 단계 | 시점 | 수익원 | 월 예상 수익 |
|------|------|--------|-------------|
| 0 | 0-6개월 | 없음 (트래픽 구축) | 0원 |
| 1 | 6-9개월 | 구글 애드센스 | 5-15만원 |
| 2 | 9-12개월 | 애드센스 + 애드포스트 | 15-40만원 |
| 3 | 12-18개월 | 입시 업체 제휴 광고 | 50-150만원 |
| 4 | 18개월+ | 프리미엄 기능 (시뮬레이터) | 100-300만원 |

### 포지셔닝: "입시 정보 서비스"가 아닌 "고교학점제 교과 설계 도구"

| 비교 | 입시 정보 서비스 | 교과 설계 도구 |
|------|-----------------|---------------|
| 경쟁자 | 진학사, 메가스터디 등 | 사실상 없음 |
| 사용 시점 | 고3 (입시 직전) | 고1 (과목 선택 시점) |
| 감정 | "합격할 수 있을까?" | "어떻게 준비할까?" |

### 킬러 시나리오 확률 분석

| 시나리오 | 확률 | 결과 |
|----------|------|------|
| A. 조용한 성공 (SEO 월 3-5만 → 제휴광고) | 35-40% | 월 100-200만 부수입 |
| B. 타이밍 히트 (2027년 검색 폭발 → 대박) | 10-15% | 월 500만+ 또는 인수 제안 |
| C. B2B 전환 (학교 도입) | 5% | Naviance 한국판 |
| D. 실패 (SEO 경쟁 실패 → 방치) | 40-45% | 시간만 손실 |

### 핵심 메시지

> "가장 큰 리스크는 기술이 아니라 인내심. SEO는 3-6개월 후에야 결과가 보이고, 그 사이 트래픽 0인 시간을 견뎌야 한다."

## 4. User Advocate (사용자 대변인): "학부모가 아닌 개발자의 시선으로 만들어진 프로토타입"

### 학부모의 진짜 니즈 계층 (우선순위순)

| 순위 | 니즈 | 현재 충족 |
|------|------|----------|
| 1 | **"이 과목 조합이면 어디 갈 수 있어?"** (역방향 검색) | 불충족 |
| 2 | **"진로가 바뀌면 어떡해?"** (시나리오 시뮬레이션) | 불충족 |
| 3 | **"우리 학교에서 이 과목 개설하나?"** (학교별 매칭) | 불충족 |
| 4 | **"A대 vs B대 같은 학과, 뭐가 다른 거야?"** (횡단 비교) | **부분 충족** |
| 5 | **"이 과목 들으면 수시/정시에서 유리해?"** (전형 연계) | 불충족 |

**현재 서비스는 순위 4번만 다루고 있으며, 가장 절실한 1~3번은 전혀 건드리지 못합니다.**

### 신뢰 문제: 가장 치명적인 구조적 약점

입시 정보 신뢰도 위계:
```
1. 대학 입학처 공식 발표 (최고)
2. 대교협/어디가 (공공기관)
3. 입시 컨설턴트 (실적으로 증명)
4. 맘카페 파워 블로거 (커뮤니티 평판)
5. 진학사/메가스터디 (브랜드)
6. 이름 없는 1인 개발 웹서비스 (최하) ← 현재 위치
```

**데이터 출처 불일치 문제**: 코드에는 "출처: ipsihogu.com"이지만, UI에는 "출처: 각 대학 입학처"로 표시. 이 괴리가 발각되면 신뢰 붕괴.

### 사용자 여정의 치명적 이탈 지점

| 이탈 지점 | 이유 | 심각도 |
|----------|------|--------|
| 홈 페이지 | **"6개 대학뿐이네? 우리 아이 목표 대학이 없어"** | 치명적 |
| 학과 검색 | "학과명을 정확히 모르겠어" (퍼지 검색 미지원) | 높음 |
| 결과 확인 후 | "우리 학교에서 이 과목을 개설하나?" | 높음 |
| 비교 모드 | 키워드 매칭 실패 시 빈 결과 | 중간 |

### "있으면 좋겠다" vs "반드시 필요하다"

| 조건 | 충족 여부 |
|------|----------|
| 이 서비스 없이는 해결 불가능한 문제 | **부분** (엑셀로 직접 정리 가능) |
| 대안이 없거나 고비용 | **불충족** (어디가/입시호구/대학 홈페이지 모두 무료) |
| 문제 해결 실패 시 비가역적 결과 | **충족** (과목 선택은 되돌릴 수 없음) |

**현재: "있으면 좋겠다" 수준. "반드시 필요하다"로 전환하려면 역방향 검색 + 시뮬레이터가 필수.**

### 학생 vs 학부모: 이중 페르소나 문제

| 역할 | 누구 | 현재 서비스 대응 |
|------|------|-----------------|
| 정보 수집자 | 학부모 (40-50대) | UI가 학부모 접근성 미고려 |
| 실제 선택자 | 학생 | 학생용 빠른 조회 모드 없음 |
| 정보 공유자 | 학부모→자녀 (카톡) | **공유 기능 전무** |

### 핵심 권장사항

1. **역방향 검색을 핵심으로 재설계** — "대학→학과→과목"이 아닌 "과목→대학/학과"
2. **2028학년도 데이터 우선 확보** — 2026 데이터는 이미 한 시즌 지남
3. **데이터 출처 투명성** — 각 항목에 출처 URL + 확인일 노출
4. **결과 공유 기능** — 카카오톡 공유, 링크 복사, 이미지 저장
5. **인문계열 데이터 추가** — 현재 자연계열만으로는 사용자 절반 이탈
6. **맘카페 인플루언서 협업** — 신뢰 문제 해결의 가장 빠른 경로

### 핵심 메시지

> "대학을 선택하고 학과를 고르면 과목을 보여준다"는 플로우를 **"내 상황을 입력하면 가능성을 보여준다"**로 뒤집는 것이 전환점.

## 5. 에이전트 간 합의/불일치 매트릭스

### 4명 모두 합의한 사항

| # | 합의 내용 |
|---|----------|
| 1 | **역방향 검색이 핵심 차별화** — 모든 에이전트가 "과목→대학" 플로우를 핵심으로 인정 |
| 2 | **현재 6개 대학은 너무 적다** — 최소 15개 대학 필요 |
| 3 | **데이터 출처 투명성 필수** — ipsihogu.com 출처 불일치 해결 필요 |
| 4 | **1인 사이드 프로젝트 프레임이 맞다** — VC 투자 기준 적용은 부적절 |

### 의견이 갈리는 사항

| 쟁점 | Builder | Skeptic | Strategist | User Advocate |
|------|---------|---------|------------|---------------|
| **횡단 비교의 실제 수요** | 시장 공백 = 기회 | 아무도 안 하는 이유 있음 | SEO 자산으로 가치 있음 | 진짜 니즈는 비교가 아님 |
| **90일 계획 현실성** | 주 8.4시간 가능 | 데이터 노동량 과소평가 | GTM 우선순위 재조정 필요 | UX 재설계가 먼저 |
| **기존 플레이어 대응** | 비즈니스 모델 충돌로 복제 안 함 | 수요 부족이 진짜 이유 | 속도+깊이로 방어 | 경쟁보다 신뢰가 문제 |

## 6. 2라운드 종합 결론

### 최종 판정: "만들되, 방향을 틀어라"

**4명의 에이전트가 수렴하는 핵심 메시지:**

1. **현재 방향(횡단 비교 도구)은 "있으면 좋겠다" 수준**에 머물러 있다
2. **"반드시 필요하다"로 전환하려면 역방향 검색 + 시뮬레이터가 필수**
3. **포지셔닝을 "입시 정보 서비스"에서 "고교학점제 교과 설계 도구"로 전환**
4. **SEO가 생존의 핵심** — 1,020개 정적 페이지로 롱테일 선점
5. **신뢰 구축이 최우선 과제** — 데이터 출처 투명성 + 맘카페 인플루언서 협업
