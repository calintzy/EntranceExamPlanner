| 에이전트 | 판정 | 근거 |

| **Critic** | **있으나 잡을 수 없음** | 무료 대안(어디가, 진학사)이 너무 강력 |

## 2. 유일한 시장 기회: "횡단 비교"

**Researcher가 발견한 핵심 인사이트:**

> 현재 시장에서 "A대학 컴공 vs B대학 컴공 vs C대학 컴공의 권장과목을 한 화면에서 비교"하는 서비스는 **존재하지 않습니다.**

- 어디가: 대학별 개별 조회만 가능
- 진학사: 입시 점수 중심, 과목 비교 없음
- 입시호구: 권장과목 검색 가능하나 횡단 비교 UI 없음

**PM도 동의:** "역방향 검색(내가 들은 과목 → 지원 가능 대학)" 알고리즘이 핵심 차별화가 될 수 있음

**타이밍:** 2028 고교학점제 첫 적용 대상(현 고1)이 지금 과목을 선택해야 하는 시점. **골든타임은 2026~2027년.**

## 3. 치명적 리스크 7가지 (Critic)

| # | 리스크 | 심각도 | 대응 가능성 |
|---|--------|--------|------------|
| 1 | **데이터 유지보수 불가** - 45개 대학 연간 ~68시간 수작업 | 치명적 | AI 파이프라인으로 10시간으로 축소 가능 (Architect) |
| 2 | **법적 리스크** - 표시광고법 위반 시 18억원 과징금 선례 | 높음 | "참고자료" 면책조항 필수 |
| 3 | **기존 플레이어 진입장벽** - 2~3주 내 복제 가능 | 높음 | 선점 + UX 차별화로 대응 |
| 4 | **수익화 벽** - 무료 대안 존재 시 유료 전환 난이 | 높음 | B2B(학교 대상) 모델 검토 |
| 5 | **하드코딩 아키텍처** - `course-data.ts`에 데이터 직접 삽입 | 중간 | 데이터 레이어 분리 1~2주 소요 |
| 6 | **데이터 시의성** - 2026 데이터가 이미 노후화 | 중간 | 2028 데이터 선점 전략 |
| 7 | **1인 번아웃** | 중간 | 커뮤니티 위키 모델 전환 |

## 4. 기술적 판정 (Architect)

### 현재 상태 진단

- `course-data.ts`에 6개 대학 데이터가 하드코딩 → **확장 불가 구조**
- `raw_recommendations.json`에 데이터 품질 이슈 (잘린 문자열, trailing comma)
- 그러나 JSON 스키마, 타입 정의 등 **기초 설계는 양호**

### 권장 로드맵

| Phase | 작업 | 기간 | 효과 |
|-------|------|------|------|
| 1 | 데이터-코드 분리 | 1~2주 | 확장 가능한 구조 확보 |
| 2 | JSON 데이터 → 프론트엔드 연결 | 2~3주 | 45개 대학 수용 가능 |
| 3 | AI 파이프라인 (PDF 파싱) | 3~4주 | 연간 업데이트 68시간 → 10시간 |
| 4 | 규모 확장 (SQLite 등) | 필요시 | 100개 이상 대학 대응 |

## 6. 종합 결론

**"가능성은 있지만, 현재 방향 그대로는 안 된다."**

### 살아남으려면 반드시 필요한 3가지

1. **"내가 듣는 과목 → 어디 지원 가능?" 역방향 검색** — 이것만이 기존 서비스와 차별화되는 유일한 기능
2. **데이터 자동화 파이프라인** — 수작업 유지보수는 1인 프로젝트에서 지속 불가
3. **법적 면책 구조** — "참고자료" 명시, 입시 결과 보장 문구 절대 금지

### 가장 현실적인 피봇 옵션

| 옵션 | 난이도 | 수익성 | 추천도 |
|------|--------|--------|--------|
| A. 횡단 비교 특화 무료 도구 + 광고 | 낮음 | 낮음 | ★★★ (MVP로 적합) |
| B. B2B — 고교 진학상담실 대상 | 중간 | 높음 | ★★★★ (US Naviance 모델) |
| C. AI 챗봇 (RAG 기반 입시 Q&A) | 높음 | 중간 | ★★★ (기술 차별화) |
| D. 커뮤니티 위키 + 데이터 크라우드소싱 | 중간 | 낮음 | ★★ (데이터 유지 해결) |

### 핵심 메시지

맘카페 34편이 증명한 건 "정보에 대한 갈증"이지, "이 앱에 대한 수요"가 아닙니다. 갈증을 해소하는 **유일한 방식**을 찾아야 합니다. 그것이 **"횡단 비교"**입니다.
